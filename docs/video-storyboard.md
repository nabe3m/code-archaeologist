## 画面収録手順書（収録前チェックリスト付き）

### 事前準備

1. **新しい削除 PR 用に demo-repo を確認**: PR #5 以外の open PR / `code-archaeologist/*` ブランチが残っていれば削除（`gh pr list --repo nabe3m/demo-repo`）。収録中の監査で新しい PR が生まれるのが見せ場
2. デモ URL にアクセスし、1回発掘を空振り実行してキャッシュとインスタンスを温める（コールドスタート回避）
3. ブラウザ: ウィンドウ 1600×900 程度 / ブックマークバー非表示 / 通知オフ / タブは「デモ UI」「demo-repo の orders/api.py」の2つだけ
4. demo-repo タブで `orders/api.py` を開いておく（オープニング用）

### 収録手順（カット別）

- **カット1（課題提起）**: demo-repo の `orders/api.py` L15 を表示 → ターミナルで `git blame orders/api.py -L 15,15` を実行して「1行しか分からない」を見せる
- **カット2（発掘）**: デモ UI で質問はプリセットのまま **⛏️ 発掘する**。タイムラインが流れ切って回答が出るまでノーカットで収録（約 40–60 秒 → 編集で 1.5〜2 倍速に）
- **カット3（出典クリック）**: 回答内の `[3]`（PR #2）をクリック → GitHub の該当 PR が開くところまで
- **カット4（監査）**: **⚖️ 監査する** → 判決カードが2枚出る: ①sleep(3) は「理由が失効 — 削除可能」→「削除 PR を作成しました」リンクをクリック → PR 本文（判定理由と証拠リスト）をゆっくりスクロール ②決済リトライは「**現在も有効 — 維持すべき**」（消す機械ではなく見分けることを一言添える）
- **カット5（差分）**: PR の Files changed タブで `- time.sleep(3)` の赤行を見せる
- **カット6（図）**: docs/architecture.png を全画面表示

### 収録後

- 生成された余分な削除 PR は close してよい（`--delete-branch` 付き）。PR #5 は実証として残す
